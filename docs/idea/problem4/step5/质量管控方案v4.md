# D3数据集质量管控方案（v4）
## 1. 问题
盲审专家要求：标签纠错、错误检测、置信度过滤等处理没有考虑。

## 2. 置信度分级机制
D1中每个样本存在两个独立标注来源：

| 信号 | 来源 | 噪声来源 |
| --- | --- | --- |
| 原始标签 | 非专业人士标注 | 非专业人士知识偏差 |
| 工作流标签 | Dify RAG+LLM | LLM推理偏差 |


两者独立意味着犯错模式不同。当两个独立信号一致时，标签可靠性应显著高于不一致时。D2实证验证了这一点：

| 置信层 | 条件 | D2占比 | 正确率 | 错误率 |
| --- | --- | --- | --- | --- |
| 高置信 | 原始标签 = 工作流标签 | 56.25% (621/1104) | 97.91% (608/621) | 2.09% |
| 低置信 | 原始标签 ≠ 工作流标签 | 43.75% (483/1104) | 87.16% (421/483) | 12.84% |


高/低置信错误率相差10.75pp，双信号一致性构成天然的置信度分级。

## 3. D3置信度标记与抽样验证
对D3的57,391条数据标记置信度，分布与D2一致：

| 置信层 | D3占比 | 样本量 |
| --- | --- | --- |
| 高置信 | 58.07% | 33,327 |
| 低置信 | 41.93% | 24,064 |


为验证D3各层实际质量，按22类 × 置信度交叉分层抽样，高置信抽440条、低置信抽330条，人工校对后：

| 置信层 | 抽样acc | 与D2同层对比 |
| --- | --- | --- |
| 高置信 | 96.59% (425/440) | D2高置信97.91%，差距1.32pp |
| 低置信 | 82.12% (271/330) | D2低置信87.16%，差距5.04pp |


### 3.1 TOST等价性检验
对高/低置信层分别进行TOST等价性检验（Δ=3%, α=0.05），以D2同层准确率作为参考基准：

**高置信层：TOST通过**

| 项目 | 值 |
| --- | --- |
| 观测acc | 96.59% |
| 90% CI | (95.17%, 98.01%) |
| 等价区间 | (94.91%, 100.91%) |
| T1 p值 | 0.0258 |
| T2 p值 | 0.0000 |
| 结论 | 90% CI完全落在等价区间内，D3高置信acc与D2统计等价 |


**低置信层：TOST未通过**

| 项目 | 值 |
| --- | --- |
| 观测acc | 82.12% |
| 90% CI | (78.65%, 85.59%) |
| 等价区间 | (84.16%, 90.16%) |
| T1 p值 | 0.8333 |
| T2 p值 | 0.0001 |
| 结论 | CI下界78.65% < 等价下界84.16%，D3低置信acc与D2不等价 |


高置信层通过TOST，说明其质量与D2高置信一致；低置信层TOST未通过，且CI下界远低于等价下界，说明D3低置信层质量显著低于D2基准，不具备等价性。

## 4. 对专家要求的回应
### 4.1 错误检测
通过双信号一致性建立高/低置信度分级。D3中24,064条低置信样本（41.93%）被标记为潜在错误区域，完成错误检测。

### 4.2 置信度过滤
过滤依据：

1. D2实证表明低置信层错误率（12.84%）远高于高置信层（2.09%），低置信本身是错误高风险区域
2. D3抽样验证低置信acc=82.12%，TOST未通过，确认D3低置信质量比D2低置信（87.16%）更差
3. 82.12%的准确率远低于D2整体基准93.21%，不具备作为训练数据的可用性

因此过滤D3低置信样本，保留高置信层（acc=96.59%，TOST通过，与D2高置信统计等价）。

过滤后：D3保留33,327条高置信样本，丢弃24,064条低置信样本。

### 4.3 标签纠错
57,000条逐一人工纠错不现实。改为对低置信层（工作流预测错误集中区域）进行badcase分析，挖掘类间混淆模式与错误共性，指导工作流优化（如补充system prompt、扩充RAG知识库），从系统层面纠正标注偏差。

# D3数据集质量验证方案：基于TOST等价性检验
## 1. 问题
D3数据集（57391条）由工作流自动标注，需验证其准确率与D2基准（1104条全量专家标注，acc=93.2%）无显著差异。

## 2. 为什么用TOST而非传统检验
| 方法 | H₀ | 通过结论 | 问题 |
| --- | --- | --- | --- |
| 传统检验 | p_D3 = p_D2 | "无法拒绝相等" | 不能证明等价，只能说不确定 |
| TOST | |p_D3 - p_D2| ≥ Δ | "差异 < Δ" | 主动证明等价，给出差异上界 |


传统检验不拒绝H₀ ≠ 证明相等（absence of evidence ≠ evidence of absence）。硕士论文需主动证明等价，TOST是唯一正确框架。

## 3. 方法
### 3.1 抽样
+ **分层**：按22类医疗器械分类目录分层，每类随机抽30条，共660条
+ **不做高低置信分层**：高/低置信是本工作流的特定产物（依赖原始标签），无普适性；22类是数据集固有分类体系，无需额外论证
+ 每类30条保证各类均有代表，总样本量满足TOST的power要求

### 3.2 TOST等价性检验
+ **参考基准**：D2全量专家标注acc = 93.2%（已知值，非估计值）
+ **等价边界 Δ = 3%**：D2内部高/低置信子集正确率差达10.8%，Δ=3%不足该差异的1/3，属保守选择
+ **显著性水平 α = 0.05**
+ **检验逻辑**：

```plain
H₀: |p_D3 - 93.2%| ≥ 3%    （不等价）
H₁: |p_D3 - 93.2%| < 3%    （等价）

等价边界: (90.2%, 96.2%)
D3的90% CI 完全落在该区间内 → 拒绝H₀ → 统计等价成立
```

### 3.3 为什么用90% CI
TOST = 两个单侧检验，各用α=0.05：

+ 检验1：p_D3 > 93.2% - 3%（左边界）
+ 检验2：p_D3 < 93.2% + 3%（右边界）

两个各5%合起来对应90% CI，这是TOST的标准做法，不是降低标准。

### 3.4 精度预估
n=660, p≈0.93时：

```plain
SE ≈ √(0.93×0.07/660) ≈ 1.1%
90% CI 半宽 = 1.645 × 1.1% ≈ 1.8%
```

TOST通过条件：观测acc > 93.2% - 3% + 1.8% = 92.0%，即观测准确率不低于92%即可通过。

## 4. TOST使用条件验证
| 条件 | 要求 | 本场景 | 结论 |
| --- | --- | --- | --- |
| 独立性 | 样本互不影响 | 发明专利之间无关联 | ✓ |
| 正态近似 | np≥5, n(1-p)≥5 | n=660, p≈0.93 → 614/46 | ✓ |
| 随机抽样 | 样本代表总体 | 22类分层随机抽样 | ✓ |
| 参考值已知 | 基准为总体真值 | D2全量专家标注，非抽样估计 | ✓ |


四个条件均满足，单样本TOST适用。

## 5. 预期结论
若TOST通过：

> 采用TOST等价性检验（Δ=3%, α=0.05），D3与D2整体准确率差异的90%置信区间为[X%, Y%]，完全落在等价边界(90.2%, 96.2%)内，拒绝原假设（p<0.05），表明两者统计等价，D3准确率与D2的差异不超过3个百分点。
>

## 6. 补充说明
+ **不先看数据再选Δ**：数据驱动选假设会膨胀第一类错误，Δ必须在抽样前论证并固定
+ **n=660的精度极限**：最小可证明的Δ ≈ 1.8%（CI半宽），Δ=1%在本样本量下不可行，需n≈1700
+ **高低置信的定位**：仅在问题背景中说明数据存在异质性，不作为方法论的一部分

# 标注质量管控方案

## 1. 背景

盲审专家提出：数据集的自动标注缺乏"标签纠错、错误检测、置信度过滤"等处理。

本项目的标注流程为：基于Dify工作流（RAG + LLM）对约57,000条医疗器械专利进行22类自动标注。工作流在专家校对的1,104条评估集（d2）上达到93.21%准确率、92.2% Macro F1。但对于剩余约57,000条标注数据（d3），缺乏质量管控机制。

## 2. 数据集结构

| 数据集 | 规模 | 标签来源 | 人工校对 |
|--------|------|----------|----------|
| d1 | ~58,000 | 原始标签（IPC映射） | 无 |
| d2 | ~1,104 | d1的分层抽样 | 有（校对标签） |
| d3 | ~57,000 | d1 - d2 | 无 |

- d2与d3的JS散度很小（论文已论证），即类别分布接近
- d2有"一级产品类别（校对）"作为真值，d3没有

## 3. 已排除的方案

| 方案 | 排除原因 |
|------|----------|
| 投票一致性检测 | 10轮投票Kappa≈1，即使拉满温度仍有98%的5/5一致性。LLM"自信地犯错"，一致性无法检测系统性偏差 |
| BERT-TextCNN交叉验证 | 模型在标注数据上训练，属于循环论证 |
| BGE-KNN近邻一致性 | BGE-KNN在d2上仅61%准确率，自身不可靠，无法用于检测标签错误 |
| 验证LLM | 无时间成本，且LLM作为baseline准确率<75%，不可靠 |
| 仅保留原始=工作流的数据 | 5个类别不足200条（放射治疗199、有源植入192、患者承载100、消毒灭菌96、输血透析42），数据集无法支持模型训练；且丢弃的恰好是分类边界模糊的难样本，造成选择偏差 |

## 4. 可用信号

d1中每个样本存在两个独立标注来源：

| 信号 | 来源 | 说明 |
|------|------|------|
| 原始标签 | IPC子组映射 | 基于专利IPC分类号规则映射到22类，有噪声但独立于LLM |
| 工作流标签 | Dify RAG+LLM | 基于分类目录知识库+LLM推理，有噪声但独立于IPC映射 |

两者独立的含义：原始标签的噪声来源是IPC映射规则的粗粒度，工作流标签的噪声来源是LLM的推理偏差，两者犯错模式不同。

## 5. d2上的实证结果

在d2（有校对标签）上验证"两信号一致性"与"标签正确性"的关系：

| 层级 | 条件 | 样本数 | 占比 | 错误率 |
|------|------|--------|------|--------|
| 高置信 | 原始标签 = 工作流标签 | 621 | 56% | 2.09% |
| 低置信 | 原始标签 ≠ 工作流标签 | 483 | 44% | 12.84% |

**核心发现**：两个独立信号一致时，标签可靠性显著高于不一致时（2% vs 13%）。这构成天然的置信度分级机制。

## 6. 方案设计

### 6.1 核心思路

既然无法穷举d3中的错误样本，也无法对57,000条逐一人工校对，则采用**抽样推断**：

> d2的错误率已知 → 从d3分层抽样校对 → 统计检验d3与d2错误率是否一致 → 推断d3整体准确率

### 6.2 置信度分级

对d3的每条数据标记置信度：
- **高置信**：原始标签 = 工作流标签（d3中约占57.8%）
- **低置信**：原始标签 ≠ 工作流标签（d3中约占42.2%）

### 6.3 分层抽样

从d3的高置信层和低置信层**分别**进行分层抽样（按22类分层）。

**每轮抽样量**：每层每类5条 → 高置信22×5=110条，低置信22×5=110条，共220条/轮

**轮次**：3轮，共660条人工校对

选择3轮的理由（基于统计模拟）：

| 轮次 | 总校对量 | 高置信95%CI宽度 | 低置信95%CI宽度 | 说明 |
|------|----------|----------------|----------------|------|
| 1轮 | 220条 | [-0.58%, 4.76%] | [6.59%, 19.09%] | CI太宽，高置信可能包含0 |
| 2轮 | 440条 | [0.20%, 3.98%] | [8.42%, 17.26%] | 可用，但精度一般 |
| 3轮 | 660条 | [0.55%, 3.63%] | [9.23%, 16.45%] | 精度较好，推荐 |
| 5轮 | 1100条 | [0.89%, 3.29%] | [10.04%, 15.64%] | 最精确，但标注量大 |

### 6.4 统计检验

校对完成后，对高置信层和低置信层分别进行**单样本比例检验**：

- H0: d3该置信层的错误率 = d2已知错误率
- H1: d3该置信层的错误率 ≠ d2已知错误率

若p-value > 0.05，不拒绝H0，即d3与d2错误率一致。

### 6.5 结论推导

若统计检验不拒绝H0：
1. d3高置信层错误率 ≈ d2的2.09%
2. d3低置信层错误率 ≈ d2的12.84%
3. d3加权准确率 ≈ 57.8%×(1-2.09%) + 42.2%×(1-12.84%) ≈ 93.4%

→ **d3数据集的标注准确率约93%，与d2一致**

## 7. 对专家三个要求的回应

| 专家要求 | 本方案的回应 |
|----------|-------------|
| 置信度过滤 | 通过双信号一致性建立置信度分级（高/低），量化各层错误率 |
| 错误检测 | 原始标签≠工作流标签即标记为低置信，44%的样本被识别为潜在错误区域 |
| 标签纠错 | 通过分层抽样验证推断整体错误率；对低置信高混淆类别可针对性优先复查 |

## 8. 执行步骤

1. 对d3生成置信度标记（原始=工作流 → 高置信，否则 → 低置信）
2. 从高/低置信层各按22类分层，每类抽5条，生成3轮共660条校对样本
3. 人工校对660条（在"校对标签"列填写22类之一）
4. 运行统计验证脚本，计算各层错误率、95%置信区间、比例检验p值
5. 在论文3.4节后新增"标注质量管控"小节，报告置信度分级机制与抽样验证结果

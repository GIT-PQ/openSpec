# BM25 超参数说明

## 分类逻辑

### 文档粒度

分类目录共 1701 条三级条目，每条三级条目作为一篇独立文档参与 BM25 索引。文档内容取 `完整结构` 列（三级序号/产品类别 + 产品描述 + 预期用途 的拼接文本）。

### 检索与聚合

一条专利（标题+摘要）作为 query，BM25 对 1701 篇文档各返回一个得分。由于同一一级类别下包含多个三级条目，需要聚合为 22 个一级类别得分后取最高者作为分类结果。

```
专利 query
    │
    ▼
BM25 检索 → 1701 个得分（每个三级条目一个）
    │
    ▼
按一级类别分组
    │
    ├── 有源手术器械（45条三级条目）→ 45 个得分
    ├── 无源手术器械（101条三级条目）→ 101 个得分
    ├── ...
    └── 临床检验器械（56条三级条目）→ 56 个得分
    │
    ▼
聚合策略
    │
    ├── max: 每组取最大值 → 22 个 max 得分 → 取其中最大值对应的类别
    │
    └── sum: 每组取求和   → 22 个 sum 得分 → 取其中最大值对应的类别
    │
    ▼
分类结果
```

**举例：** 假设"无源植入器械"下有 306 条三级条目，"医疗器械消毒灭菌器械"下有 57 条：

- **max 策略：** 从 306 个得分中取 max，从 57 个得分中取 max，与其他 20 个类别的 max 一起共 22 个值，取其中最大的作为分类结果
- **sum 策略：** 从 306 个得分中取 sum，从 57 个得分中取 sum，与其他 20 个类别的 sum 一起共 22 个值，取其中最大的作为分类结果

两种策略的本质差异：max 只看"有没有一个子类高度匹配"，sum 则叠加"所有子类的匹配信号"。因此 sum 天然对三级条目多的类别有利（306 个得分求和 vs 57 个得分求和，基数差异大），而 max 消除了条目数量的影响。

## BM25 公式

$$\text{score}(D, Q) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot (1 - b + b \cdot \frac{|D|}{\text{avgdl}})}$$

其中 $f(q_i, D)$ 为词 $q_i$ 在文档 $D$ 中的词频，$|D|$ 为文档长度，$\text{avgdl}$ 为平均文档长度。

---

## 参数介绍

### k1 — 词频饱和参数

- **含义：** 控制词频对得分的影响上限。词频越高得分越高，但增长逐渐平缓（饱和效应）。
- **取值范围：** 通常 [0.5, 3.0]，经典默认值 1.5。
- **直觉：**
  - k1 越小 → 词频很快饱和，出现1次和出现多次的词得分差异小，文档排名更依赖"是否有该词"而非"该词出现几次"
  - k1 越大 → 词频饱和慢，高频词获得更高权重，适合长文档中词频差异对相关性影响大的场景
- **本实验搜索值：** [0.5, 1.0, 1.5, 2.0, 2.5]

### b — 文档长度归一化参数

- **含义：** 控制文档长度对得分的惩罚力度。
- **取值范围：** [0, 1]，经典默认值 0.75。
- **直觉：**
  - b = 0 → 完全不惩罚文档长度，长文档和短文档同等对待
  - b = 1 → 完全按文档长度归一化，长文档被强惩罚
  - b 越大 → 长文档中出现的词被更多稀释，短文档中的匹配更有价值
- **本实验搜索值：** [0.3, 0.5, 0.75, 0.9, 1.0]

### agg — 类别聚合策略（本实验自定义）

- **含义：** BM25 以三级条目为文档粒度检索，每条专利对同一一级类别下的多个三级条目各有一个得分，需要聚合为一个类别得分。
- **选项：**
  - **max：** 取该类别下所有三级条目的最高得分。含义——只要有一个子类强匹配即可，偏向精确命中
  - **sum：** 取该类别下所有三级条目的得分之和。含义——该类别下多个子类都相关则加分，偏向覆盖面广的类别
- **本实验搜索值：** [max, sum]

## 搜索结果摘要

50 组超参数组合（5 k1 × 5 b × 2 agg），最佳 top-5：

| 超参数 | Accuracy | Macro F1 |
|--------|----------|----------|
| k1=2.5, b=0.9, agg=sum | 0.2998 | 0.1383 |
| k1=2.5, b=1.0, agg=sum | 0.2989 | 0.1409 |
| k1=2.0, b=1.0, agg=sum | 0.2971 | 0.1371 |
| k1=1.5, b=0.9, agg=sum | 0.2971 | 0.1365 |
| k1=2.5, b=0.5, agg=sum | 0.2971 | 0.1358 |

**规律：**

- **sum 聚合全面优于 max**：accuracy 高 2-5 个百分点。原因是医疗器械分类目录中，同一一级类别下的三级条目数量差异大（如"无源植入器械"306条 vs "医疗器械消毒灭菌器械"57条），sum 聚合能利用条目数量的信息。
- **高 k1 + 高 b 在 sum 下表现最好**：k1=2.0-2.5, b=0.9-1.0。高 k1 让词频差异更充分体现（分类目录原文中术语重复出现是有意义的），高 b 让短条目（如简单的三级分类描述）中的匹配更有价值。
- **max 聚合的 macro F1 反而更高**（0.19-0.22 vs 0.13-0.14），说明 max 对少数类更友好，不会因类别条目数多而偏向大类别。

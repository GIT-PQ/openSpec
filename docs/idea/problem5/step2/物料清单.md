## 输出文件规范

实验产出两个文件，通过 `patent_id` 关联：

### 1. 嵌入数据集（机器用）

- **格式：** parquet（跨平台，列式压缩）
- **路径：** `work/data/patent_embeddings.parquet`
- **内容：**

| 列名 | 类型 | 说明 |
|------|------|------|
| patent_id | str | 专利唯一标识 |
| title | str | 专利标题 |
| abstract | str | 专利摘要 |
| true_label | str | 真实标签（人工标注/最终确认） |
| original_label | str | 专利原始标签 |
| bm25_label | str | BM25预测标签 |
| llm_zs_label | str | LLM zero-shot预测标签 |
| knn_bge_label | str | KNN+BGE预测标签 |
| rag_label | str | RAG+规则方法预测标签 |
| embedding | list[float] | BGE嵌入向量（1024维） |

- **用途：** KNN计算、指标评估，代码自包含，无需回xlsx关联

### 2. 标注对比表（人可读）

- **格式：** xlsx
- **路径：** `work/data/annotation_comparison.xlsx`
- **内容：**

| 列名 | 说明 |
|------|------|
| patent_id | 专利唯一标识 |
| title | 专利标题 |
| abstract | 专利摘要 |
| true_label | 真实标签 |
| original_label | 专利原始标签 |
| bm25 | BM25预测标签 |
| llm_zero_shot | LLM zero-shot预测标签 |
| knn_bge | KNN+BGE预测标签 |
| rag | RAG+规则方法预测标签 |

- **用途：** Excel中直接对比、筛选、案例分析；论文中挑典型案例

### 3. 分类目录向量库（方法③专用）

- **格式：** parquet
- **路径：** `work/data/catalog_vectors.parquet`
- **内容：**

| 列名 | 类型 | 说明 |
|------|------|------|
| text | str | 拼接后的分类目录条目（品名+描述+用途） |
| label_idx | int | 类别索引（0-21） |
| label | str | 类别名称 |
| embedding | list[float] | BGE嵌入向量（1024维） |

## 补充说明

- **待标注数据**是最关键的，三个方法都依赖它
- **LLM信息**需与你现有RAG标注方法保持一致（同一模型），否则对比实验变量不唯一
- **GPU**不是必须的，CPU也能跑，只是慢一些
- parquet和xlsx通过 `patent_id` 关联，parquet面向代码计算，xlsx面向人工查看和论文分析

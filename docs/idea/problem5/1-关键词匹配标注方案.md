# 关键词匹配标注方案（BM25 Baseline）

## 目标

作为实验对照组①，验证纯关键词匹配方法在医疗器械专利分类上的表现，为论文"方法对比"章节提供基线数据。

## 方法概述

```
输入: 专利文本（标题 + 摘要）
      │
      ▼
┌──────────────────────────────────────────┐
│  Step 1: 领域分词                         │
│  关键词 → jieba 自定义词典                │
│  确保"超声手术刀"不被切成"超声/手术/刀"   │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  Step 2: BM25 检索                        │
│  22篇文档 = 分类目录原文（品名+描述+用途） │
│  专利文本作为 query → BM25 检索            │
└──────────────┬───────────────────────────┘
               │
               ▼
  对22个类别打分 → 取 top-1 作为分类结果
```

**核心设计：** 关键词提取的成果不直接作为BM25文档内容，而是作为**jieba自定义词典**提升分词质量。BM25的文档内容来自分类目录原文，更自然地反映类别语义。

## 物料

| 物料 | 来源 | 用途 |
|------|------|------|
| 22类关键词 | `work/data/医疗器械关键字_lite.json` | 加载为jieba自定义词典，提升专业术语分词质量 |
| 医疗器械分类目录 | `work/data/医疗器械分类目录.xlsx` | 每个类别的品名举例+产品描述+预期用途拼接为BM25文档 |
| 待标注数据 | ~1100条专利 | 含标题、摘要、原始分类标签 |

## 关键词的双重角色

```
关键词提取的成果
    │
    ├──→ jieba 自定义词典（本方案）──→ 提升分词质量 ──→ BM25匹配更准
    │
    └──→ （你的RAG方法中）作为检索关键词库
```

**为什么关键词做词典而非文档内容：**

- BM25的匹配质量取决于token粒度。jieba默认分词会把"射频消融导管"切成"射频/消融/导管"，丢失专业术语的完整性
- 将关键词加载为自定义词典后，"射频消融导管"作为一个整体token参与匹配，语义更精确
- 分类目录原文（品名+描述+用途）天然是高质量的类别描述文档，比关键词拼接更自然、信息更完整

## 实现步骤

### Step 1: 构建jieba自定义词典

```python
import json
import jieba

# 加载关键词
with open("work/data/医疗器械关键字_lite.json", "r", encoding="utf-8") as f:
    kw_data = json.load(f)

# 收集所有关键词，去重后写入自定义词典文件
all_keywords = set()
for cat_name, cat_data in kw_data["categories"].items():
    all_keywords.update(cat_data["product_names"])
    all_keywords.update(cat_data["category_terms"])
    all_keywords.update(cat_data["description_terms"])
    all_keywords.update(cat_data["usage_terms"])

# 写入jieba词典格式：词语 词频 词性
dict_path = "work/data/bm25_custom_dict.txt"
with open(dict_path, "w", encoding="utf-8") as f:
    for kw in sorted(all_keywords):
        f.write(f"{kw} 5 n\n")  # 词频5，词性n(名词)

# 加载自定义词典
jieba.load_userdict(dict_path)
print(f"自定义词典加载完成：{len(all_keywords)} 个术语")
```

### Step 2: 构建BM25索引（分类目录原文）

```python
import pandas as pd
from rank_bm25 import BM25Okapi

# 22个类别（与模型LABELS顺序一致）
LABELS = [
    '中医器械', '临床检验器械', '医用康复器械', '医用成像器械', '医用诊察和监护器械', '医用软件',
    '医疗器械消毒灭菌器械', '口腔科器械', '呼吸、麻醉和急救器械', '妇产科、辅助生殖和避孕器械',
    '患者承载器械', '放射治疗器械', '无源手术器械', '无源植入器械', '有源手术器械', '有源植入器械',
    '注输、护理和防护器械', '物理治疗器械', '眼科器械', '神经和心血管手术器械', '输血、透析和体外循环器械', '骨科手术器械'
]

# 读取分类目录，每个sheet对应一个类别
xls = pd.ExcelFile("work/data/医疗器械分类目录.xlsx")
corpus = []

for label in LABELS:
    doc_parts = []
    for sheet_name in xls.sheet_names:
        if label in sheet_name:
            df = pd.read_excel(xls, sheet_name=sheet_name, header=0)
            for _, row in df.iterrows():
                # 拼接品名举例、产品描述、预期用途
                for col in ['品名举例', '产品描述', '预期用途']:
                    val = row.get(col, '')
                    if pd.notna(val) and str(val).strip():
                        doc_parts.append(str(val).strip())
            break

    doc_text = " ".join(doc_parts) if doc_parts else label
    corpus.append(doc_text)

xls.close()

# 分词（使用加载了自定义词典的jieba）
tokenized_corpus = [list(jieba.cut(doc)) for doc in corpus]

# 构建BM25索引
bm25 = BM25Okapi(tokenized_corpus)
print(f"BM25索引构建完成：{len(corpus)} 个类别文档")
```

### Step 3: 对专利文本分类

```python
def classify_patent_bm25(title: str, abstract: str, bm25_index, labels) -> dict:
    """
    使用BM25对专利文本进行分类

    Args:
        title: 专利标题
        abstract: 专利摘要
        bm25_index: BM25索引
        labels: 类别标签列表

    Returns:
        dict: 包含预测类别、top-5类别及分数
    """
    query_text = f"{title} {abstract}"
    tokenized_query = list(jieba.cut(query_text))

    scores = bm25_index.get_scores(tokenized_query)

    # 获取排序后的类别
    sorted_indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)

    result = {
        "pred_label": labels[sorted_indices[0]],
        "pred_score": float(scores[sorted_indices[0]]),
        "top5": [
            {"label": labels[idx], "score": float(scores[idx])}
            for idx in sorted_indices[:5]
        ]
    }

    return result
```

### Step 4: 批量标注并评估

```python
# 加载待标注数据
df = pd.read_json("待标注数据路径")  # 或 csv

results = []
for _, row in df.iterrows():
    pred = classify_patent_bm25(row["title"], row["abstract"], bm25, LABELS)
    results.append({
        "patent_id": row.get("id", ""),
        "title": row["title"],
        "abstract": row["abstract"][:100],  # 截断保存
        "true_label": row["label"],
        "pred_label": pred["pred_label"],
        "pred_score": pred["pred_score"],
        "top5": pred["top5"]
    })

# 保存结果
result_df = pd.DataFrame(results)
result_df.to_csv("bm25_classification_results.csv", index=False, encoding="utf-8-sig")

# 评估
from sklearn.metrics import classification_report, accuracy_score
print(f"Accuracy: {accuracy_score(result_df['true_label'], result_df['pred_label']):.4f}")
print(classification_report(result_df['true_label'], result_df['pred_label']))
```

## 评估指标

| 指标 | 说明 |
|------|------|
| Accuracy | 整体准确率 |
| Precision/Recall/F1 (per class) | 每类别的精确率/召回率/F1 |
| Macro-F1 | 宏平均F1（处理类别不平衡） |
| Weighted-F1 | 加权F1 |

## 预期问题与应对

| 问题 | 原因 | 应对 |
|------|------|------|
| 短文本匹配差 | 专利摘要太短，BM25匹配词少 | 合并标题+摘要+权利要求（如有） |
| 跨类别关键词重叠 | 如"超声"同时出现在手术器械和成像器械 | BM25的IDF机制自然处理，无需额外干预 |
| 某类别分类目录条目少 | 分类目录中某些类别条目少 | 记录该类别文档长度，分析时注意 |
| 专业术语被错误切分 | jieba默认词典不包含医疗器械术语 | 自定义词典已解决，但需检查覆盖率 |

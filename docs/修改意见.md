# 修改意见
1. 第一章国内外研究现状调研文献偏旧（仅两篇较新），前沿不足。文献综述凌乱，未梳理本课题的几个前沿方向，如专用预训练模型，层次文本分类，长文本分类等相关工作。对任务难点分析不深入。建议补充 2023–2026 专利分类、长文本分类、垂直领域分类最新进展。另外，研究现状的总结未能明确指出当前领域存在的关键问题与研究缺口（均为陈旧的通用问题），导致研究动机不够充分。建议立足本课题的研究痛点，如，分类目标不匹配，标签质量问题，层次结构未被利用等。
2. 本文研究内容描述为“构建数据集、做模型、做系统“工程化流程，没有针对具体的关键技术问题的学术上改进，创新性显不足。研究流程图1.1，是科普介绍吗？没有任何技术逻辑链。整体方法显薄弱，技术含量低。
3. 第二章整体以知识点介绍为主，本章存在意义很小，介绍的方法在本课题中的使用不足与局限性（后续方法改进的动机）均未描述。章节内容逻辑较为割裂，专利分类、传统机器学习、深度学习、大模型技术之间缺乏内在关联与过渡。对专利专用预训练模型、层次文本分类、长文本分类等主流方法的相关技术基本未介绍。建议强化评述性、补充前沿方法、提升综述的学术性与逻辑性。
4. 第三章，数据筛选规则不够严谨，缺少去重机制与数据统计分析，数据代表性与纯净度论证不足。数据集的自动标注主要依赖 LLM，但未做 “领域适配”，没有在垂直领域上微调，虽有RAG 知识库支持标注，但知识单薄（只有《分类目录》，缺少法规、指导原则、审查指南等）。另外，标签纠错、错误检测、置信度过滤等处理也没有考虑。
5. 实验部分本文标注方法只和“原始标签”比，不和 “其他标注方法“比较，无法证明论文方法优势。缺少”案例分析“。
6. 用LLM改写少数类文本，扩充数据。专利文本的严谨性是否能随便生成？是否会改变技术内容，本部分没有经过专家校验。
7. 第四章提出的 BERT-TextCNN 是2018–2020 年的经典组合结构，不是新模型。因此本方法仅属于简单组合，无原创性结构创新与机制创新，理论深度与学术贡献不足。
8. 医疗器械分类目录具有明确层级结构（1，2，3级），本文方法未采用层次分类，违背目录结构，问题定义上实用性较弱。
9. 对比实验中，对比基线严重陈旧LR、KNN、RF、XGBoost，单独 BERT、单独 TextCNN，没有对比近年专利分类/文本分类优秀方法。无法证明方法的优秀性。消融实验做了数据增强消和模型模块消融，缺少最关键的“层次信息”，“领域知识”等消融。外部验证集数据量不大且来源单一（同源），无法证明泛化能力和工业可用性。
10. 第五章，需求系统功能完整性不足，仅实现单条专利文本分类，未支持批量上传、批量分类、结果导出等实际业务高频，工程性较弱。未表述系统的非功能需求。整体流程缺少”系统测试与验证“，开发流程不完整。
11. 本文口语化、冗余表述较多；图表不太规范。部/分图片里文字较小，表格尽量不分页（或用续表拆分），建议全文格式再修改。

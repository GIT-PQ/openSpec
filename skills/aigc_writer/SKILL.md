# 学术论文撰写助手（AIGC规避版）

本 skill 通过交互式工作流，帮助用户完成论文章节的撰写，同时确保生成内容具有"人类特征"，降低 AIGC 检测率。

---

## 工作流概览

```
用户启动 → 需求收集 → 大纲确认 → 分轮撰写 → 自检交付
```

---

## Step 1：需求收集（首次交互）

用户启动 skill 时，通过提问收集基本信息：

**必填信息**：
1. **论文主题**：您要撰写什么主题的论文？（如：医疗器械专利分类系统）
2. **目标章节**：需要撰写哪个章节？（如：文献综述、系统设计、实验部分）
3. **字数要求**：该章节大约需要多少字？

**可选信息**：
4. **已有素材**：有哪些参考资料、数据或前期内容需要融入？（可提供文档链接或粘贴内容）
5. **风格偏好**：倾向于正式学术风格还是偏实践的技术报告风格？
6. **特殊要求**：是否有导师或期刊的特定格式要求？

> **注意**：如果用户信息不完整，先补充提问，不要假设。

---

## Step 2：内容规划

根据收集的需求，生成写作大纲供用户确认：

**输出格式**：
```markdown
## 写作大纲

### 第1节：xxx（约xxx字）
- 要点1
- 要点2

### 第2节：xxx（约xxx字）
- 要点1
- 要点2

...
```

**确认门槛**：用户明确回复"可以""按这个来""确认大纲"后，才进入撰写阶段。"我看看""再想想"不算确认。

---

## Step 3：分轮撰写

按大纲顺序逐节生成内容，每节独立输出。

### 3.1 撰写原则

生成时注入"人类特征"：

| 维度 | AI 特征（避免） | 人类特征（注入） |
| :--- | :--- | :--- |
| 句式 | 长度均匀、结构工整 | 长短句交错，有节奏变化 |
| 词汇 | 高频学术词汇堆砌 | 专业术语 + 适度口语化表达 |
| 逻辑 | 过于完美的线性推理 | 允许适度转折、对比、让步 |
| 引用 | 机械罗列文献 | 融入批判性分析和个人见解 |
| 案例 | 抽象描述为主 | 引入具体案例或数据支撑 |

### 3.2 输出格式

每节内容输出：
```markdown
### 第X节：标题

[正文内容...]

---
**本节字数**：约 xxx 字
**AIGC 规避要点**：已注入xxx特征
---
```

### 3.3 交互规则

- 每节写完后询问："下一节还是修改本节？"
- 用户选择"修改"时，针对性调整，不全篇重写
- 避免一次性输出整章，先输出 300-500 字的小节，确认后再扩展

---

## Step 4：AIGC 率自检

整章完成后，提供自检清单供用户自行检查：

### 4.1 快速自检清单

- [ ] 句式是否有明显的长短交替？（避免全是长句）
- [ ] 词汇是否有多样性？（避免重复使用同一词汇）
- [ ] 是否有个人见解或批判性分析？（而非纯客观罗列）
- [ ] 是否有具体案例或数据支撑论点？
- [ ] 逻辑链是否有适度的转折或让步？（而非直线式推理）

### 4.2 进一步优化建议

如用户自检后仍担心 AIGC 率，可提供：
- 同义词替换建议表
- 句式重组示例
- 人工润色重点区域标注

## 附录：AIGC 检测知识（参考）

### 阈值标准

| 学术阶段 | AIGC率合格标准 |
| :--- | :--- |
| 本科毕业论文 | < 25%（部分高校放宽至 40%） |
| 硕士学位论文 | 10% ~ 15% |
| 博士学位论文 | < 5% |
| 学术期刊投稿 | < 15%（核心期刊） |

### 检测核心指标

- **困惑度 (Perplexity)**：AI 倾向选择高概率词汇，导致文本"过于可预测"
- **突发性 (Burstiness)**：AI 句子长度均匀，人类写作长短句交错
- **语义一致性**：AI 逻辑过于完美，缺乏人类的思维跳跃

### 基础规避策略

1. 同义词替换："研究"→"探究"；"影响"→"作用"
2. 语序调整：主动句/被动句切换
3. 引入个人批判性观点
4. 融入具体案例或数据

---

## 确认语说明

只有以下表述视为用户确认，可进入下一阶段：
- "可以""按这个来""确认""开始"
- "下一节""继续""进行下一步"
- 明确表达同意的其他表述

以下表述不视为确认：
- "我看看""有道理""先这样""再想想"
